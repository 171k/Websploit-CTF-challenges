Understanding HTTP Headers

A deep dive into how the web communicates behind the scenes.

HTTP headers are metadata sent with every HTTP request and response. They carry key-value pairs that give browsers and servers extra context about what's being transferred.

Types of Headers

  • General Headers: Apply to both request and response messages.
  • Request Headers: Sent by the client (e.g. User-Agent, Accept).
  • Response Headers: Sent by the server (e.g. Server, Set-Cookie).
  • Entity Headers: Provide information about the body (e.g. Content-Type).

Example: Response Headers

HTTP/1.1 200 OK
Content-Type: text/html
Content-Length: 3056
X-Powered-By: Express
X-Flag: ? 😉

Inspecting HTTP Headers

You can view HTTP headers in several ways:

  • 📦 Using browser dev tools (F12 → Network tab → click a request → Headers)
  • 🧰 Using terminal tools like: 
    curl -I https://example.com
  • 🌐 Using browser extensions like “Live HTTP Headers”

Custom Headers

Developers often use X-* headers for custom purposes, like debugging, tracking, or even easter eggs 😏

🔎 Fun Facts

  • The infamous X-Clacks-Overhead: GNU Terry Pratchett header is a tribute to author Terry Pratchett (source).
  • Some CDNs inject headers like X-Cache: HIT to indicate caching behavior (MDN).
  • Custom headers like X-Debug or X-Beta-User are often used internally by dev teams.
